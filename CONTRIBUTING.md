# Contributing to AWS Solutions Architect Reference

Thank you for your interest in contributing to this repository! We welcome contributions to help improve this resource for the AWS Solutions Architect community.

---

## 🛠️ How You Can Contribute

1.  **Correct Mistakes**: Spot a technical error or a typo? Open a pull request or an issue to correct it.
2.  **Add New Concepts**: Want to write a deep-dive on an AWS service or pattern we missed? (e.g., IoT, Blockchain, or Big Data analytics patterns).
3.  **Enhance Diagrams**: Improve the readability or structure of our Mermaid architecture diagrams.
4.  **Refine Interview Q&As**: Add common interview questions or refine existing responses to be more accurate and helpful.

---

## 📝 Guidelines

*   **Keep it AWS-Focused**: Every concept and scenario must tie directly to native AWS services, tools, and best practices.
*   **Aesthetics Matter**: All diagrams must be written in clean, readable Mermaid syntax.
*   **Comprehensive Explanations**: When adding scenarios, follow our 12-section template (Problem Statement, Requirements, Key Services, Walkthrough, Cost, Alternatives, etc.) to maintain the high standards of this repository.
*   **Respect Least Privilege**: In all code examples or design walk-throughs, ensure IAM roles and security settings adhere to the principle of least privilege.
